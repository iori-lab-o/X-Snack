# コードレビュー・スキル

コードをレビューする際は、以下の手順に従ってください。

## レビュー・チェックリスト

1. **正確性**: コードは意図した通りに動作していますか？
2. **エッジケース**: エラー条件は適切に処理されていますか？
3. **スタイル**: プロジェクトの規約（TypeScript, Expoなど）に従っていますか？
4. **パフォーマンス**: 明らかな非効率性はありませんか？

## フィードバックの提供方法

- 何を変更すべきか具体的に示してください。
- 「何を」だけでなく「なぜ」必要なのかを説明してください。
- 可能な場合は代替案を提案してください。
