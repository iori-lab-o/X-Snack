---
description: 毎日の作業開始時に、Gitの状態、最近のコミット、タスクドキュメントを確認し、その日の作業計画を立てるためのワークフローです。
---

1. Gitの状態を確認します。
// turbo
```bash
git status
```

2. 直近のコミットログ（10件）を確認し、最近の変更内容を把握します。
// turbo
```bash
git log --oneline -n 10
```

3. プロジェクトのタスク管理ファイルを確認します。以下のファイルを `view_file` ツールで読み込んでください。
   - `TASKS.md`
   - `TODO.md`
   - `AGENT.md` (プロジェクト全体のコンテキスト確認用)

4. 確認した情報を元に、以下の項目を含めて本日の作業内容を提案してください。
   - **現在の状況要約**: 未コミットの変更有無や、直近で完了したタスク。
   - **確認事項**: `TASKS.md` や `TODO.md` から読み取れる次のタスク。
   - **提案**: 今日具体的に何から着手すべきか。

5. ユーザーに提案内容を確認し、承認または指示を仰いでください。
